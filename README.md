## Creating an issue

If you‚Äôve found a bug in the Web API, or have ideas on how we could improve it, please [create a GitHub issue and fill out the required information][New GitHub Issue]. It‚Äôs greatly appreciated.

We will triage the request and mark it as one of the following types:

- <kbd>[bug]</kbd> ‚Äî Unexpected behaviour relating to the Web API that can be reproduced by Spotify.
- <kbd>[suggestion]</kbd> ‚Äî Requests for additional functionality or changes in the Web API that has been acknowledged by Spotify.
- <kbd>[documentation]</kbd> ‚Äî Requests for changes to documentation that has been acknowledged by Spotify.
- <kbd>[question]</kbd> ‚Äî Relates to the Web API, but does not fit into the above categories.

Depending on the outcome, we will mark it as the following status:

- <kbd>[wontfix]</kbd> ‚Äî We will close the issue, as this isn‚Äôt expected to be considered in the next 6 months. Any progress in the future will be posted on respective issues even after it is closed.
- <kbd>[offtopic]</kbd> ‚Äî We will close this issue, as this does not relate specifically to the Web API. Where possible, we will direct you to the correct issue tracker / location.
- <kbd>[duplicate]</kbd> ‚Äî We will close the issue, as this has already been asked and to consolidate all discussions. We will reference the respective issues to guide you and any other visitors to a single issue for updates.
- <kbd>[more info needed]</kbd> ‚Äî We require more information from the reporter in order to make progress. We will close it if no response is made within 2 weeks.
- <kbd>[inactive]</kbd> ‚Äî The issue requires more information and has received no engagement after some period of time, and we will close the issue. If reporter wishes to raise issue at a later point, they can create a new issue and reference.

## Frequently asked questions

#### I have an issue with other Spotify APIs and SDKs. Can I post them here?
Unfortunately not; this issue tracker is only for the Spotify Web API. For bugs and requests related to other Spotify APIs and SDKs, please visit the respective issue trackers on GitHub:

- [spotify/web-api]
- [spotify/web-api-auth-examples]
- [spotify/web-playback-sdk]
- [spotify/ios-sdk]
- [spotify/android-sdk]
- [spotify/ios-app-remote-sdk]
- [spotify/android-app-remote-sdk]
- [spotify/android-auth]

#### I have a non-technical issue with Spotify. Is this the right place to post them?
Unfortunately not; our GitHub issue trackers do not provide support for non-technical Spotify questions. Please submit these in our official [Community Forums]. Posting them here will result in them being closed as <kbd>[offtopic]</kbd>.

#### I‚Äôm an official Spotify partner and I have a technical question. Is this the right place to post them?
This repository only deals with public-facing APIs. Partnership-specific questions don‚Äôt belong here as this is a public forum; please contact your Spotify partner manager.

#### How long does it take for my issue to be addressed?
Typically, we aim to start investigating all new issues within one week (except during holiday periods). 

#### How do I get my feature request added?
Given the scale of Spotify, not all feature requests can be added. It‚Äôs important to note that all feedback is extremely valued and whilst we have implemented many suggestions from the community, we cannot make any guarantees for suggestions.

#### Why was my issue closed?
All of the reasons issues might be closed are mentioned in [Creating an issue](#creating-an-issue) above, which will explain the whole process better.
